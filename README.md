# Color Separation Workstation

Windows-приложение для анализа PDF/PostScript/EPS, расчёта покрытия красок и просмотра цветоделений с имитацией печатного растра.

Проект ориентирован на Windows 10-11 и рабочий процесс препресса: входной файл разбирается через Ghostscript, каналы CMYK/spot превращаются в TIFF-сепарации, после чего их можно смотреть в интерфейсе с наложением красок, управлением каналами, масштабом и настройками растрирования.

## Возможности

- Принимает PDF, PS и EPS, включая уже сепарированные PostScript-файлы.
- Считает среднее покрытие по CMYK и spot-краскам.
- Сохраняет TIFF-сепарации в папку `Separation` и удаляет пустые/лишние композитные каналы.
- Показывает цветоделения в GUI с субтрактивным наложением красок.
- Позволяет включать/выключать каналы, менять порядок наложения и масштабировать предпросмотр.
- Поддерживает приближённое отображение Pantone-цветов по известным значениям и номерным семействам.
- Имитирует растр: AM, flexo AM, FM, hybrid XM и error diffusion.
- Берёт углы и параметры растра из PostScript, PPD-профилей или ручных настроек.
- Использует OpenCL/GPU там, где доступно, с fallback на CPU.
- Ведёт журнал запуска, действий пользователя, расчётов и ошибок.

## Требования

1. Windows 10 или Windows 11.
2. Python 3.11 или 3.12.
3. Ghostscript для Windows. Команда `gswin64c` должна быть доступна через `PATH`.
4. Python-зависимости из `requirements.txt`.

Установка зависимостей:

```powershell
py -m pip install -r requirements.txt
```

## Запуск

Из папки проекта:

```powershell
py app.py
```

Также можно запускать через батник:

```powershell
.\run_app.bat
```

## Работа с файлами

В интерфейсе выбираются один или несколько входных файлов `PDF`, `PS` или `EPS`, папка вывода и DPI. Результаты сохраняются в подпапку вида:

```text
Separation/temp_YYYY-MM-DD_HH-MM-SS
```

Для уже сепарированных PostScript-файлов программа читает признаки пластин, включая `%%PlateColor`, и сопоставляет страницы с `Cyan`, `Magenta`, `Yellow`, `Black` и spot-красками. Если Ghostscript отдаёт один монохромный TIFF без меток пластин, канал определяется по имени файла: `C`, `M`, `Y`, `K`, `Cyan`, `Magenta`, `Yellow`, `Black`, `Pantone` или `PMS`.

## Предпросмотр

После расчёта открывается рабочая область просмотра цветоделений:

- каналы отображаются с реальным наложением, а не отдельными плоскими слоями;
- каждый канал имеет явный чекбокс включения/выключения;
- порядок каналов меняется кнопками вверх/вниз;
- доступен масштаб, режим `1:1`, вписывание страницы и поворот;
- navigator показывает положение текущего viewport на странице.

Для CMYK используются технологические цвета. Для Pantone есть таблица известных значений и fallback по номерным семействам, чтобы spot-краски не выглядели случайными hash-цветами.

## Растрирование

В предпросмотре доступны режимы:

- `Традиционный AM`;
- `Флексо AM` с минимальной удерживаемой точкой;
- `Стохастический FM`;
- `Гибридный XM`;
- `Error diffusion`;
- режим без повторного растрирования.

Для AM/XM применяются заданная линиатура, DPI, форма точки и угол канала. Участки чистой бумаги и 100% плашки сохраняются без растровой структуры; растр применяется только к промежуточным тонам. Если входная сепарация уже выглядит как готовый limited-tone или bitonal RIP-вывод, повторное AM/FM-растрирование не применяется автоматически.

1-bit TIFF в подпапке `Bitonal` создаётся только для действительно бинарных пластин, где один бит/пиксель корректно означает наличие или отсутствие точки на форме.

## PPD-профили

Приложение автоматически ищет PPD-файлы в папке проекта и позволяет загрузить внешний профиль через интерфейс. Из профиля читаются:

- доступные разрешения;
- наборы линиатур;
- углы CMYK и spot-каналов;
- поддерживаемые формы точки.

В репозитории есть пример `BCCDI1_SPARKXT2.PPD`. Для ESKO FlexRip PPD доступны flexo-наборы углов, например `C 7.5° / M 37.5° / Y 82.5° / K 67.5°`.

## Архитектура

Проект строится как двухслойная система:

- Python остаётся оболочкой, workflow, GUI, прототипом алгоритмов и reference implementation для тестов.
- C++/Rust backend предназначен для тяжёлой обработки больших TIFF: tiles/stripes, memory mapping, многопоточность, 1-bit TIFF writer и низкоуровневый контроль памяти.

Большие формы нельзя обрабатывать как один NumPy-массив. Например, лист 600 x 400 мм при 2400 dpi даёт примерно 56,688 x 37,800 пикселей, то есть около 2.14 ГБ на один 8-bit канал без временных буферов. Поэтому production-путь должен быть тайловым или строчным.

Ключевые файлы:

- `app.py` - PySide6 GUI, управление задачей, предпросмотр и пользовательские действия.
- `inkcalc.py` - запуск Ghostscript, разбор входов, расчёт покрытия и подготовка сепараций.
- `rip_core.py` - tiled renderer предпросмотра и смешивание каналов.
- `native_backend.py` - выбор Python/native backend, env-настройки tiles/stripes/mmap и ctypes ABI.
- `halftone.py` - алгоритмы растрирования и разбор PostScript-параметров.
- `gpu_halftone.py` - OpenCL/GPU backend с fallback на CPU.
- `plate_bits.py` - сохранение бинарных пластин.
- `ppd_profiles.py` - чтение PPD-профилей.
- `app_logging.py` - настройка логирования.
- `native/` - C++ ABI-шаблон для будущего нативного renderer/backend; контракт описан в `native/README.md`.

## Логи

Журнал пишется в:

```text
Logs/app.log
```

Логируются запуск приложения, действия пользователя, этапы расчёта, проблемы Ghostscript, ошибки импортов и необработанные исключения. Файл ротируется автоматически: до 2 МБ на файл, максимум 5 архивных копий. Папка `Logs` не добавляется в Git.

## Тесты

Запуск основного набора тестов:

```powershell
python .\scripts\run_tests.py --timeout 30
```

Runner запускает каждый файл тестов отдельно, с таймаутом и принудительным завершением тестового процесса. Это важно на Windows: Qt, Pillow, OpenCL и Ghostscript могут кратковременно держать файловые handles после завершения теста.

Быстрая проверка синтаксиса:

```powershell
py -m py_compile app.py inkcalc.py halftone.py rip_core.py gpu_halftone.py native_backend.py plate_bits.py ppd_profiles.py app_logging.py
```

## Сборка EXE

После установки PyInstaller:

```powershell
py -m pip install pyinstaller
py -m PyInstaller --noconsole --onefile --name InkCoverageCalc --icon Logo.ico app.py
```

Готовый файл появится в папке `dist`. Иконка приложения берётся из `Logo.ico`.

## Ограничения

Это учебный исследовательский проект, а не сертифицированный RIP. Растрирование и цвет Pantone являются математической/визуальной имитацией для анализа и предпросмотра. Закрытые алгоритмы коммерческих RIP-систем не воспроизводятся побитово.
