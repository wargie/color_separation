# Калькулятор покрытия краской

Windows-приложение для расчёта среднего покрытия PDF/PostScript по CMYK и spot-краскам.

## Что нужно установить

1. Python 3.11 или 3.12.
2. Ghostscript для Windows. После установки команда `gswin64c` должна быть доступна в `PATH`.
3. Python-зависимости:

```powershell
py -m pip install -r requirements.txt
```

## Запуск GUI

```powershell
py app.py
```

В окне можно выбрать один или несколько файлов PDF, PS или EPS, папку вывода, DPI и запустить расчёт. TIFF-сепарации сохраняются в папку `Separation`; неиспользованные композиты и пустые каналы удаляются автоматически.

После расчёта доступен предпросмотр цветоделения с субтрактивным наложением красок: например, `Magenta` поверх `Yellow` отображается как красный. Каналы можно включать и выключать чекбоксами, менять порядок наложения кнопками `Вверх` и `Вниз`, а также приближать и отдалять предпросмотр кнопками `-`, `100%`, `+`.

В предпросмотре можно включить имитацию растра: офсетный clustered-dot AM, флексо-AM с минимальной удерживаемой точкой, стохастический FM, error-diffusion или гибридный XM. Для AM/XM-режимов программа применяет заданную линиатуру и углы каналов; для PostScript/EPS пытается взять параметры из `setcolorscreen` и `sethalftone`, а при их отсутствии использует стандартные углы CMYK.

Растрирование выполняется через OpenCL на доступном GPU с автоматическим переходом на CPU. Чистая бумага и участки со 100% краски сохраняются без растровой структуры; AM/FM/Hybrid применяются только к промежуточным тонам.

Если сепарация уже выглядит как готовая форма или limited-tone RIP-вывод, программа не применяет к ней повторное AM/FM-растрирование. 1-bit TIFF в подпапке `Bitonal` создаётся только для действительно бинарных пластин, где промежуточные уровни отсутствуют и один бит/пиксель корректно означает наличие или отсутствие точки на форме. Limited-tone TIFF остаётся серой маской покрытия, чтобы не превращать реальные 70%, 128/255 и похожие уровни в грубые 0/100; при включённом режиме растрирования такая маска может быть заново просеяна для визуальной симуляции печатной точки.

Предпросмотр построен через отдельное ядро `rip_core`: GUI передаёт список пластин, настройки растра и цветовые каналы, а ядро собирает изображение плитками. Такой renderer не держит весь лист одной гигантской матрицей и подготовлен к замене Python/NumPy-реализации на нативную DLL. В папке `native` лежит минимальный C++ ABI-шаблон `rip_core_native`, который будет расширяться под быстрый tiled rendering и viewport/loupe-пересчёт.

Программа обнаруживает PPD-профили в папке проекта и позволяет загрузить внешний PPD через интерфейс. Из профиля читаются доступные разрешения, наборы линиатур, углы CMYK/spot и поддерживаемые формы точки. Для ESKO FlexRip PPD доступны производственные flexo-наборы, например `C 7.5° / M 37.5° / Y 82.5° / K 67.5°`. Реализованные формы точки: круглая, эллиптическая, квадратная и линейная. Это математическая имитация для анализа и предпросмотра, а не воспроизведение закрытых алгоритмов Esko Classic, DynaCell или SambaFlex.

Для уже сепарированных PostScript-файлов можно выбрать сразу комплект пластин. Если один PS содержит несколько страниц-пластин, программа читает `%%PlateColor` и сопоставляет страницы с `Magenta`, `Yellow`, `Black` и другими красками. Нулевые каналы и композитный дубль не выводятся. Если Ghostscript отдаёт файл только как один монохромный TIFF без меток пластин, программа считает его отдельной пластиной и пытается определить канал по имени файла: `C`, `M`, `Y`, `K`, `Cyan`, `Magenta`, `Yellow`, `Black`, `Pantone` или `PMS`.

## Логи

Программа ведёт журнал действий пользователя, этапов расчёта, проблем запуска приложения и ошибок в файле:

```text
Logs/app.log
```

Файл ротируется автоматически: до 2 МБ на файл, максимум 5 архивных копий. Папка `Logs` не добавляется в Git.

## Сборка exe

После установки PyInstaller:

```powershell
py -m pip install pyinstaller
py -m PyInstaller --noconsole --onefile --name InkCoverageCalc app.py
```

Готовый файл появится в папке `dist`.
